### 의사결정 / 비코드 사항

- **모노레포 유지**: `content/` 는 레포 루트에 두고 `web/lib/blog.ts` 에서 `../content/blog/` 참조. Next.js `web/` 격리 + 콘텐츠가 앱 코드와 섞이지 않음.
- **Blog 스택**: MDX 대신 plain markdown (gray-matter + remark). React 컴포넌트 임베딩이 필요해지면 MDX 로 업그레이드 여지.
- **로고는 Kim 원본 PNG 존중**: 한 번 SVG 로 재해석했다가 반려됨. 원본 투명 PNG 를 크롭·설치하는 방향으로 복귀. 교훈은 memory 에 저장됨.
- **색 팔레트**: 브랜드 네이비 (로고) + 테라코타 (액션) + 크림 (배경) 공존. NYT 식 네이비 마스트 + 컬러 액션 패턴.
- **이미지 저작권**: PhotoPlaceholder 로 전 페이지 실제 이미지 없이도 "사진 준비중" 명시. 더술닷컴 원본 이미지 사용 금지 원칙 유지.

### 알게 된 것 / 삽질

- **Postgres upsert + ON CONFLICT**: `onConflict` 대상 컬럼에 UNIQUE 제약이 없으면 에러. CLAUDE.md §7 원본 스키마에 없어서 2회에 걸쳐 마이그레이션 추가. **다음부턴 스키마 설계 시 ETL upsert 키도 함께 고려**.
- **Postgres upsert 배치 내 중복 금지**: 같은 batch 안에 같은 (conflict key) pair 가 두 번 오면 "cannot affect row a second time" 에러. xlsx 에 크롤러 중복이 있어 320 그룹 완전 중복 발견 → payload 에서 미리 dedupe 로직 추가.
- **Supabase 키 신포맷**: `sb_publishable_*` / `sb_secret_*` 이 legacy JWT 를 대체. env var 이름은 여전히 `*_ANON_KEY` / `*_SERVICE_ROLE_KEY` 로 유지 (코드 호환성).
- **shadcn Button v3+ 는 `asChild` 대신 `render` 프롭** (base-ui 기반). 현재는 `buttonVariants` 를 Link 에 직접 적용하는 방식으로 우회.
- **체커보드 무늬가 실제 픽셀로 export 되는 경우 존재**: 디자인 툴 UI 상의 투명 표시기가 export 파일에 박힘. Kim 이 재export 해서 해결.
- **PIL `getbbox()`**: 투명 PNG 의 투명 여백 자동 감지·크롭. 로고 headers 크기 조정에 필수.
