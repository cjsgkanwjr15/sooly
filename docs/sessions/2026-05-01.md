## 오늘 한 것

### 큰 덩어리 6개

1. **Google OAuth** — 매직 링크에 더해 Google 로그인 추가. `signInWithGoogle` server action +
   LoginForm 의 Google 버튼 (공식 멀티컬러 G 마크). 기존 `/auth/callback` 라우트는 PKCE code
   exchange 를 이미 처리해서 코드 변경 없음. Kim 이 Google Cloud Console 에서 OAuth client
   발급 + Supabase Auth Providers 에 Client ID/Secret 입력. dev 에서 즉시 동작 확인.
2. **프로필 편집 풀세트** — `/settings/profile` 페이지 + `ProfileEditForm` + `updateProfile`
   server action (zod). 표시 이름·사용자명 (2~20자, 영문 소문자·숫자·언더스코어, 시작 제한 없음
   = `13wogns` 같은 숫자 시작 OK)·한 줄 소개 편집. username 변경 시 `/u/{new}` 자동 redirect.
   예약어 (`admin`, `settings`, `login` 등) 차단. unique 충돌 (Postgres 23505) 감지.
3. **프로필 통계 카드** — `/u/[username]` 에 통계 섹션 추가. **카테고리별 분포** (가로 막대) +
   **양조장 TOP 3** (1·2·3 뱃지 + 양조장 페이지로 클릭) + **별점 분포** (★5~★1 히스토그램) +
   **최근 30일 활동** (Stat 행). 통계용 lightweight 쿼리 (모든 체크인) + 그리드용 (60 limit)
   분리. application 레벨 집계 — Phase 1 규모(체크인 수십~수백)에선 충분, 추후 트래픽 늘면
   materialized view.
4. **양조장 페이지 라인업 보강** — 빈약한 텍스트 카드 → **이미지 + 별점 + 카테고리·도수·용량**
   풀 카드 그리드. 1/2/3 컬럼 반응형. 별점 자동 정렬 (별점 desc → 체크인 수 desc → 가나다순).
   별점 집계는 양조장 제품 IDs 한 번에 `.in()` 으로 N+1 회피.
5. **카카오 OAuth — 시도 후 숨김** — `signInWithKakao` action + 노란 카카오 버튼까지 만들었으나
   **새 Kakao Developers UI 가 개인 앱한테 OAuth Redirect URI 등록 자체를 막아놨음**. 비즈
   앱 인증 (사업자 정보 + 1~3일) 받기 전엔 동작 X. 코드는 주석 블록으로 보존, 비즈 인증 후
   5분 만에 부활 가능.
6. **블로그 ko/en 이중언어 시스템** — 파일 명명 `{slug}.{locale}.md` 강제 (`welcome.ko.md` +
   `welcome.en.md`). `getAllPosts(locale)` + `getPostBySlug(slug, locale)` 가 locale 매칭,
   없으면 다른 언어 fallback + 상단 배너 안내 ("이 글은 영어로만 작성되어 있어요"). 리스트에선
   fallback 글에 `KO` / `EN` 배지. `availableLocales` 가 둘이면 "다른 언어판 있음" 안내.
   Schema.org `inLanguage` 추가. **welcome.en.md 본문 작성** — 한국어 톤 (회사 소개 + 시장
   진단 + 겸손) 따라 자연스럽게 영문화.

### 의사결정 / 비코드 사항

- **카카오 OAuth = 비즈 인증 받은 후로 미루기**: 새 Kakao Developers UI 가 개인 앱 OAuth Redirect
  URI 등록을 차단. 비즈 앱 인증 (사업자 정보 + 며칠 심사) 필요. 코드는 보존 (주석) + 버튼 숨김.
- **사용자명 시작 문자 제한 X**: `^[a-z][a-z0-9_]*$` (영문 시작 강제) → `^[a-z0-9_]+$` (시작 제한
  없음) 으로 풀음. 자동 생성된 username (`13wogns_502d` 처럼 숫자 시작) 도 사용자가 수정 가능
  해야 함. trigger (0004) 와 form 검증 정책 일치.
- **`/settings` 가 인덱스 + `/settings/profile` 가 sub**: 향후 알림·계정 삭제 등 추가될 자리.
  user-menu 는 인덱스로 진입 (discoverability ↑), `/u/[username]` 의 "프로필 편집" 은 직접
  편집 페이지로 (자주 쓰는 액션 fast-path).
- **블로그 파일명 = `{slug}.{locale}.md` 강제**: 다른 옵션들 (`/ko/welcome.md` 디렉토리, frontmatter
  분리 등) 대비 가장 명확. 한국·영어 둘 중 하나만 작성된 글도 자연 표현 가능 + 같은 slug 페어링
  자동.
- **Fallback 표시 정책**: 영문판 없으면 (a) 안 보임 vs (b) 한국어로 표시 + 배너. **(b) 채택** —
  EN 사용자도 글 자체는 발견 가능, 미공개보다 fallback 이 나음. ai-translate-blog (Phase 2 별도
  세션) 끝나면 fallback 자연 해소.
- **welcome.en.md 톤 = "Sooly, a global hub for Korean alcohol"**: 입니다 체 → 자연스러운
  formal English (not stiff). hedge ("not a flashy beginning, but…") 유지. 핵심 용어
  (jeontongju, makgeolli, cheongju) 는 romanization + 부연 (italics 첫 등장 시).

### 알게 된 것 / 삽질

- **Next 16 `redirect()` 외부 URL 지원**: Server Action 안에서 `redirect(googleAuthorizeUrl)`
  처럼 외부 도메인 redirect 가능. 검색해보고 확인. `node_modules/next/dist/docs/01-app/03-api-reference/04-functions/redirect.md`
  에 "redirect also accepts absolute URLs and can be used to redirect to external links"
  명시.
- **Supabase Auth Site URL 갱신 필요**: 4-25 도메인 연결 때 Redirect URLs 만 추가했고 Site URL
  은 `sooly.vercel.app` 그대로. OAuth fallback 일부 케이스에서 vercel.app 으로 빠질 수 있어
  Kim 한테 `https://sooly.co.kr` 로 변경 안내. 매직 링크는 코드에서 emailRedirectTo 명시해서
  영향 X 였지만 정리.
- **Kakao Developers 새 UI = 개인 앱 OAuth 막음**: 카카오 로그인 → 일반 페이지에 OAuth Redirect
  URI 칸 자체가 없음. 30분 헤맸음. 답: 비즈 앱 인증 안 받은 개인 앱은 OAuth 못 씀. **다음 세션
  이전에 비즈 인증 신청 추천**.
- **Supabase nested select 가 array 또는 object** 둘 다 줄 수 있음: `products(...)` 같은
  단일 FK 인데도 `Array<...>` 로 타입 추론될 때 있음. `unwrap<T>(v: T | T[] | null): T | null`
  헬퍼로 정규화. 이미 product detail 에서 쓰던 패턴, 이번엔 /u/[username] 통계용 nested
  select 에서도 동일.
- **양조장 별점 집계 = `.in()` 한 번**: N+1 회피. 양조장당 제품 ~50개 미만이라 한 쿼리로
  전체 체크인 가져와 application 집계. Phase 2 에 트래픽 늘면 RPC 또는 materialized view.
- **gray-matter + remark + reading-time 파이프라인**: 블로그 빌드 시 frontmatter 분리 →
  remark-gfm + remark-html → reading-time 분 계산. 새 blog.ts 도 이 패턴 그대로, slug 파싱
  로직만 추가.
