### 의사결정 / 비코드 사항

- **`.com` 포기 + `.co.kr` 단독**: `sooly.com` 이미 점유, `sooly.kr` 도 점유. `sooly.co.kr` available 하고 한국 소비자 대상에 충분. 글로벌 확장 시점에 `.io` / 다른 TLD 재검토.
- **Workspace 보류, Free Gmail 시작**: $6+/mo per seat 는 매출 전 단계에선 오버. soolyhello@gmail.com 으로 Search Console·GA·Instagram·기타 통합. 매출 생기면 Workspace 마이그레이션.
- **GA 게이트는 NODE_ENV**: VERCEL_ENV 까지 보지 않음. Preview 배포도 production 통계로 들어오지만 Kim 본인 테스트 트래픽이라 무시할 만한 수준 + 로컬 dev 노이즈만 차단되면 충분.
- **`VERCEL_PROJECT_PRODUCTION_URL` 우선**: `VERCEL_URL` 은 deployment-specific 해시라 sitemap/robots 호스트로 부적합. 빌드마다 호스트가 바뀌면 SEO 신호 분산. Stable production URL 우선 사용.
- **Schema.org 알코올**: schema.org 에 알코올 음료 전용 type 없음. `Product` + `additionalProperty` 로 ABV/Volume 표현. `Brewery` (LocalBusiness 서브타입) 는 존재하므로 양조장은 그대로 사용.

### 알게 된 것 / 삽질

- **Vercel 새 UI 의 env var 위치**: 별도 "Environment Variables" 메뉴가 사라지고 Settings → Environments → Production 안에 nested. 직관적이지 않아 Kim 이 헤맸음. 다음에 다른 사람 가이드할 때 주의.
- **Framework Preset 자동 감지 실패**: 모노레포(Root Directory = `web`) 구조에서 Vercel 이 Next.js 못 알아채는 경우 있음. 빌드는 성공하지만 출력이 정적 파일로 처리되어 모든 URL 이 404. Settings → Build and Deployment 에서 명시적으로 Next.js 선택해야 함.
- **Deployment Protection "Standard"**: 무료 플랜 기본값이 모든 배포(Production 포함)에 인증 장벽. Production 도메인은 404 로 응답, 배포-specific URL 은 401. Kim 처럼 "공개 사이트" 가 목적이면 Disabled 로 변경해야 함.
- **Vercel 빈 커밋 push = 가장 깔끔한 fresh deploy 트리거**: env 변경 후 캐시 충돌 의심될 때, "Use existing Build Cache" 해제 redeploy 로 안 풀리면 빈 커밋 푸시가 가장 확실.
- **Korean ccTLD 는 Korean registrar 에서만**: Namecheap 에서 `sooly.kr` 검색 시 "Unsupported TLD". `.kr` / `.co.kr` 은 가비아·hosting.kr 등 한국 업체 통해서만 등록 가능.
- **API 키 채팅 노출 = 즉시 burned**: Kim 이 새 service_role 키를 채팅에 한 번 붙여 넣으면 그 키는 다시 로테이션 대상. 무한 루프 회피하려면 Vercel 대시보드에 *직접* 붙여넣기 + Kim 이 값을 채팅에 안 보내는 패턴 필수.
- **JSON-LD escape**: React 의 `<script>` text content 자동 escape 가 `<` 를 살려둬 JSON-LD 가 깨질 수 있음. `dangerouslySetInnerHTML` + `</script>` 방어용 `<` → `<` 치환이 안전한 패턴.
- **Vercel new UI Settings → Domains 메뉴 부재**: Vercel 새 UI 에서 "Domains" 는 사이드바에 별도 항목 없고 Environments 의 Production 안에서 같이 관리. 익숙해지면 일관됨.
