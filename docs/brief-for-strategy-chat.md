# Sooly — 전략 상담 브리프 (Claude 웹용)

> 이 문서는 Claude (claude.ai) 와 같은 외부 AI에게 붙여넣고
> 프로젝트 방향·우선순위·비즈니스 전략에 대해 질문할 때 쓰는 self-contained 요약이다.
> 코드 구현 디테일보다 제품·시장·의사결정 맥락에 집중.

## 1. 한 문장 정의

**Sooly** 는 한국 전통주·막걸리·소주·과실주를 국내외 사용자가 **발견·기록·공유**할 수 있는 정보 허브 + 양조장 대상 B2B SaaS. **Untappd(맥주)·Vivino(와인)의 한국술 버전**.

- **장기 목표: 본인이 직접 양조장 차려서 술 만들기.** Sooly 는 그 전 단계 자산(브랜드·트래픽·업계 네트워크) 구축용

## 3. 시장 상황 (중요한 맥락)

### 경쟁사/레퍼런스
| 회사 | 모델 | 규모 |
|------|------|------|
| 술담화 (한국) | 전통주 구독 박스 | 2022년 매출 70억 |
| 데일리샷 (한국) | 주류 스마트오더 앱 | MAU 130만 |
| 술마켓 (한국) | 전통주 이커머스 | 인스타 11K |
| **Untappd (미국)** | 맥주 체크인 SNS | 유저 **900만**, 체크인 **10억**, **B2B SaaS 20,000+ venues** |
| **Vivino (덴마크→미국)** | 와인 마켓플레이스 | 유저 **5천만**, 와인 1,200만 |

### 한국 주류 규제 (매우 중요)
- **전통주만 온라인 통신판매 합법** (2017년~)
- 일반 주류는 스마트오더(픽업)만, 자택 배송 불가
- **전통주 = 정보허브 + 마켓플레이스 결합이 합법적으로 가능한 유일한 주종**

### 시장 빈틈
1. 한국 전통주용 **Untappd/Vivino 포지션 공석**
2. 양조장 1,300곳 대부분 **IT·마케팅 인력 부재** → B2B 기회
3. **영문 한국술 정보 절대 부족** → 해외 한인 1,000만 + K-culture 팬 시장 선점 기회

## 4. 오늘까지 만든 것 (웹사이트 기능 단위)

### 데이터베이스
- **더술닷컴 크롤링** → 제품 **791종** + 양조장 **431곳** 실제 적재
- 지역 매핑 100%, 카테고리(탁주·약주·청주·증류주·과실주·리큐르) 정규화 완료
- 크롤러 중복 수집분 자동 제거, 수동 주소 보정 6건

### 사용자 페이지 (모두 한/영 이중언어 토글 지원)
- **홈** — Hero + 카테고리 6개 그리드 + 오늘의 발견 + Featured Brewery + 블로그 티저
- **제품 리스트** — 검색창, 카테고리·지역 chip 필터
- **제품 상세** — Vivino 스타일 2컬럼 (이미지 + 타이틀·통계·별점CTA), 맛 프로필 radar, 비슷한 제품, 양조장 이야기
- **카테고리 랜딩** 6개 (`/categories/탁주` 등) — 가이드 + 마시는 법 + 대표 양조장 + 제품
- **양조장 리스트 + 상세** — 지역 필터, 제품 수 집계
- **블로그 (Journal)** — markdown 기반, SEO 최적, 첫 포스트 "Sooly 를 시작하며"
- **B2B 랜딩 /for-breweries** — Verified 플랜(Phase 3) 티저 + 리드 수집
- **양조장 공식 콘텐츠 슬롯** — 제품·양조장 상세에 "양조장이 직접 전하는 이야기" 점선 박스 placeholder + "공식 등록 문의" CTA (→ /for-breweries 연결). 실제 CMS 는 Phase 3 Verified 플랜 때 활성화, 지금은 UI 자리만 예약해 **양조장이 사이트 방문 시 "여기가 내 공간" 을 인지**하도록.

### 정보 아키텍처 (6-layer 모델)
술에 관한 정보를 다음 6 레이어로 분리해 각각 다른 출처·톤·권한으로 관리:
1. **사실 데이터** (도수·용량·지역) — 크롤링 자동, ✅ 완료
2. **큐레이션 맛 노트** (Sooly 에디토리얼) — AI 정제 + 편집, ⏳ 다음 세션
3. **양조장 공식 이야기** (B2B) — 양조장 직접 작성, 📦 UI 슬롯만 준비
4. **양조장 소식·뉴스** (B2B) — 양조장 직접 업데이트, 📦 UI 슬롯만 준비
5. **유저 체크인 + 리뷰** (UGC) — 로그인 유저, 📦 Empty state 준비됨
6. **커뮤니티 Q&A** — Phase 2+

UGC(유저 체크인) ≠ 양조장 공식 글 ≠ Sooly 에디토리얼 — **각각 시각적 힌트 다르게**
(Verified 뱃지, 점선 박스, 유저 아바타) 로 신뢰도·출처 명시.

### 디자인·브랜딩
- 로고: 투명 배경 PNG, 한/영 자동 전환 (주전자+사발 일러스트 + SOOLY 워드마크)
- 팔레트: 네이비(로고) + 테라코타(액션) + 크림(배경)
- 타이포: Fraunces(영문 세리프) + Gowun Batang(한글 세리프) + Inter(본문)
- 배경에 은은한 주전자 실루엣 워터마크

### 인프라
- Next.js 16 + Supabase(Postgres/Auth/Storage) + Vercel 예정
- SEO 기본 구조 (SSR/ISR, meta tags)
- 모노레포: `web/` + `scripts/(파이썬 크롤러+Node ETL)` + `content/blog/` + `supabase/migrations/`

## 7. 핵심 의사결정 원칙

- **"부끄러울 정도로 빨리 공개"** — 완벽주의 거부. MVP 1~2주 내 배포
- **이미지 저작권**: 더술닷컴 이미지 사용 금지. 플레이스홀더 → Kim 직접 촬영 → 양조장 공식 제공 순서로 교체
- **텍스트 저작권**: 더술닷컴 제품 설명은 AI 리라이팅 후 사용, 사실 데이터(도수·용량)는 그대로
- **만들지 않을 것**: 네이티브 앱 (12개월 후), 복잡한 추천 알고리즘 (Phase 2+), 자체 결제, 자체 배송·재고
- 유저 10만 기다리지 말고 **유저 100명부터 수익 실험 시작**
- **정보 출처를 섞지 않는다** — 유저 UGC, 양조장 공식 콘텐츠, Sooly 에디토리얼은 DB 스키마·UI 표현·권한 모두 분리. 한 번 섞이면 다시 나누기 어렵고 신뢰도 훼손.
- **B2B 리드 수집은 Day 1 부터** — Verified 플랜 자체는 Phase 3 지만 양조장이 사이트 방문 시 바로 문의할 수 있게 /for-breweries 랜딩·제품 상세 슬롯·푸터 링크 준비 완료. 론칭 직후부터 리드 쌓기 시작.

## 8. 지금 Kim 의 고민거리

다음은 결정해야 하거나 외부 조언이 있으면 좋은 항목들:

### A. 브랜딩·도메인 (아직 미확정)
- 서비스 이름 후보: **Sooly** (1순위), Soolmap, Hansool, Jumak, Nuruk
- `.com` / `.io` 도메인 확보 필요
- KIPRIS 상표 검색 (제32·33류 주류 카테고리) 안 함
- 인스타그램 핸들 확보 여부 미확인

### B. 론칭 타이밍
- MVP 기능 어디까지 넣고 공개해야 할까?
- Untappd/Vivino 는 **체크인 기능 없이는 의미 없는** 플랫폼. 인증+체크인이 최소 공개 조건?
- 아니면 정보 허브로 먼저 공개하고 체크인은 나중에 추가?

### C. 첫 100명 획득 전략
- Kim 본인 인스타(400명) 기반으로 초기 유저 유치?
- 네이버 카페 / 디시인사이드 전통주 갤러리 등 커뮤니티 침투?
- 인플루언서(바텐더·술 리뷰어)에게 씨딩?
- 유료 광고는 아직 불필요?

### D. B2B 세일즈 접근 (Phase 3 건데 미리 리드 쌓아야)
- 양조장 직접 콜드 아웃리치? (메일·전화·방문)
- "찾아가는 양조장" 사업단 같은 정부기관 연계?
- 술 박람회·시음회 참여로 네트워킹?
- Phase 1 공개 후 트래픽 데이터 보여주며 영업?

### E. 한/영 이중언어 시점
- 지금 UI는 한국어 기본, 로고만 이중화
- **본격 영문 전환 시점**: AI 번역으로 DB 영문 필드 채우는 때? 해외 마케팅 시작할 때?
- `sooly.kr` 과 `sooly.com` 같은 도메인 분리? 아니면 같은 도메인에서 `/en/` 경로?

### F. 직접 제조 전환 시점 (Kim 의 장기 목표)
- 플랫폼 트래픽이 어느 수준 되면 Kim 이 직접 만든 술 론칭?
- 주류제조면허는 최소 1년 준비 필요 → **언제 시작?**
- 양조 기술·파트너 확보 루트?

### G. 콘텐츠 전략
- 블로그 누가 쓰나? Kim 혼자? AI 도움받기? 외부 필진 섭외?
- 주당 몇 편? SEO 효과 보려면 어느 정도 필요?
- 어떤 주제부터? (양조장 방문기 / 입문 가이드 / 페어링 / 인터뷰 중)

### H. 윤리·사회적 책임
- 주류 플랫폼의 **미성년자 차단** 의무 (연령 확인 로직 필요)
- **과음 조장 이슈**: 체크인으로 "많이 마실수록 좋다" 가 되지 않게 설계?
- 특정 양조장·제품만 편애해 보이지 않도록 투명성 장치?

### I. B2B 콘텐츠 인벤트리브(기여 동기) 설계
- 양조장이 **왜** 자기 콘텐츠를 Sooly 에 직접 써야 하는가?
  - 인스타그램·네이버 블로그 이미 하는데 또 Sooly 에 쓸 유인이 약함
- Verified 플랜 월 9~29만원을 낼 근거: **유료 고객이 받는 가치 구체화 필요**
  - 대시보드 데이터? 신제품 노출? 유저 직접 DM? 리뷰 대응 도구?
- 무료 티어도 있어야 할까? (공식 이야기만 1개 쓸 수 있는 free, 뉴스피드·통계는 paid)
- 양조장 입장에서 **"다른 채널에서 안 되는 것"** 이 Sooly 에 뭐가 있는지 명확히 해야 컨버전 가능

### J. 정보 우선순위: UGC vs 에디토리얼 vs B2B
- 같은 제품 상세 페이지에 유저 리뷰 3개·Sooly 에디토리얼·양조장 공식 글이 공존할 때 **무엇을 먼저 보여줄지**?
- Vivino 는 유저 리뷰 우선, Wine Folly 는 에디토리얼 우선, Untappd 는 유저 체크인 우선
- Sooly 는 **초창기(체크인 축적 전)엔 에디토리얼 + 양조장 공식** 을 앞에 두고, 체크인 N건 쌓이면 유저 쪽이 위로 올라가는 단계적 구조?
