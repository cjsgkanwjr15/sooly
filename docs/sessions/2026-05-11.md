### 의사결정 / 비코드 사항

- **DB array i18n 톤 정책 = 카테고리·6축·지역 매핑과 일관**: 한국 고유 어휘 = romanization +
  부연, 보편 = 영어. "정제수" = "Water" (간결).
- **블로그 정책 = .ko.md canonical / .en.md derived**: 사후 hand-edit 비권장, `--force` 에
  덮어씀. 수정은 .ko.md 에서 하고 force 재생성.
- **명함 vs cohort outreach 시퀀스 분리**: 명함 = 메모리 trigger + 진정성 자산 hook. 1기
  cohort ask 같은 큰 ask 는 follow-up 단계. "지난 [엑스포명] 에서 인사드렸던 Sooly 김재훈입니다"
  시퀀스로 cold mail 답장률 5~10배 ↑.
- **트래픽 권위 self-doubt 의 답 = 메시지 reframe**: 트래픽 기다리기 = 닭과 달걀, 1년 후도
  "충분한 트래픽" 기준 못 채울 가능성. 자산 (카탈로그 800/431 + 한·영 이중언어 + Kim 본인
  IG 진정성) 이 트래픽보다 더 강한 무기.
- **양조장 사장님 50명 = 45 잊음 + 3~5 슬쩍 + 1~2 메모리 + 0~1 자발 컨택**: 명함 돌리는
  *진짜 목표* = 그 1~2명을 위한 작업. 50명 모두 awe 어필 X.

### 알게 된 것 / 삽질

- **Length-mismatch fail rate ~16%**: AI 가 비슷한 entry 합침 (`["쌀(국내산)", "쌀(이상국내산)"]`
  같은 중복성). dirty data 의 경우 길이 보존 정책이 강제로 압축 만듦 — UX 측엔 오히려 OK
  지만 fail rate 가 높음. 다음 세션 retry 시 prompt 에 "각 entry 독립 번역" 명시 권장.
- **5-07 entry 의 "4년 IG" 잘못 기록 = CLAUDE.md 자가 검증 필요성**: Claude 가 자기 메모리
  (CLAUDE.md) 만 보고 판단하면 자가 강화. Kim 의 fact-check 없으면 다음 세션도 잘못 인용.
  *관련 사실은 항상 Kim 한테 한 번 더 확인* 룰.
- **Canva > 자체 SVG**: 30분짜리 명함 디자인 = 디자이너의 영역. 내가 SVG 직접 짜면 *작동하는
  정보 카드* 까지는 가능하지만 *짜치지 않은 명함* 까지는 어려움. PNG raster 로고 임베드 +
  시스템 폰트 의존 = 인쇄 품질·일관성 약함. Canva 한국 명함 템플릿이 압도적 효율.
- **Python `qrcode` 라이브러리 = SVG path-based output OK**: viewBox 25×25 cell-based path
  로 vector QR 생성. 명함 SVG 안에 inline embed → 인쇄 무손실.
- **명함은 양조장 사장님 *기억 trigger* 가 핵심 기능**: 자랑·hard sell·IT 용어 다 회피. "한국술
  좋아해서" 진정성 voice + 카탈로그 사실 + "양조장 페이지 봐주실래요" 부탁 = 가장 자연스러운
  pattern.
- **`@sooly_hello` IG 콘텐츠 시드 = 엑스포 직후 일제 업로드 액션 아이템**: 명함 → 사이트 →
  follow-up 메일 → Sooly IG follow 시퀀스에서 *follow-up 메일 시점* 에 콘텐츠 채워져 있어야
  함. 엑스포 다녀온 직후 (월~수) 5~10 게시물 일제 업로드.
