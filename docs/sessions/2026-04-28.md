### 알게 된 것 / 삽질

- **Next.js 16: `middleware.ts` → `proxy.ts` 리네임**: AGENTS.md 가 "Read docs first"
  경고했고 실제로 deprecated. 함수명도 `proxy`. 같은 `NextRequest`/`NextResponse` 타입.
  matcher config 동일.
- **Supabase Auth 의 Site URL fallback 동작**: emailRedirectTo 가 Redirect URLs
  화이트리스트에 없으면 *Site URL 로 fallback*. 결과 — Site URL 이 `localhost:3000` 인 채로
  배포하면 prod 매직 링크가 localhost 로 리다이렉트. **Site URL + Redirect URLs 둘 다
  세팅 필수**, sooly.co.kr 붙을 때마다 같이 갱신.
- **Gemini Free tier 한도 + 503 capacity spike**: free 가 5 RPM. paid 활성화해도 503 자주.
  같은 모델 retry 보다 **fallback chain (2.5-flash → flash-lite → 2.0-flash)** 통과율 ↑.
  Lite 가 가장 안정.
- **`gemini-2.0-flash` 가 신규 사용자에 unavailable (404)**: "no longer available to new users".
  fallback chain 마지막 모델은 `gemini-2.0-flash-001` 또는 `gemini-1.5-flash` 로 교체 권장.
- **API 키 채팅 노출 = 즉시 burned (반복 패턴)**: Supabase service_role 2회·Gemini 1회.
  **메모리 entry 추가** (`feedback_secret_in_chat.md`) — 다음 세션부터 키 받기 *직전* 에
  사전 차단.
- **가비아 DNS CNAME 값에 점 (.) 강제**: FQDN 표준. `cname.vercel-dns.com` → 거부,
  `cname.vercel-dns.com.` → 통과. A 레코드는 무관.
- **Search Console 속성이 다른 Google 계정에 있을 때**: 새 계정에서 안 보임. 계정 확인 필수.
  옛 계정에 새 도메인도 verify 시키면 두 계정 모두 운영 가능 (verification meta 태그
  array 로 여러 개 동시 노출).
- **`form action` 에 client wrapper 패턴**: server action 자체는 redirect/revalidate 만.
  client state (editing toggle 등) 정리는 wrapper 함수에서 `await action(formData);
  onSuccess?.()` 식. 인라인 수정 폼이 저장 후 자동 닫힘.
- **`--sample N --dry-run` 으로 sample 검증**: AI 번역 1,251 row 전체 돌리기 전 5개 샘플
  로 톤 확인. ROI 큼.
